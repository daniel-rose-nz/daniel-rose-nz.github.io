Javascript + DOM

Feet vs Socks / HTML vs CSS

Hypertext markup language (HTML) and Cascading style sheets(CSS) work together to display content in an apporpriate form. HTML is a markup language that provides simple formating capabilities directly to content within a document, it also tags and categorises content which allows it be targeted by CSS. CSS provides powerful stylistic features that allow a designer to control how content is displayed, basically allows control of the form of the content. CSS can be applied within a single document or be referenced by multiple html files to give consistency across an entire multi-page website. Having styles controlled from a single CSS file allows for efficient manipulation of styles and negates the need to update individual files when changes are desired.

I suppose one might use the analogy that if a body part(content) was categorised or tagged(HTML) as a "feet", then choosing the appropriate fashioinable socks would be the CSS.

Control Flow is Cool

The control flow is the order in which the computer executes statements in a script.

Control flow is cool because you can control the energy flow through the software.

Control flow means that when you read a script, you must not only read from start to finish but also look at program structure and how it affects order of execution. Code is run in order from the first line in the file to the last line, unless the computer runs across the (extremely frequent) structures that change the control flow, such as conditionals and loops. 

                
              if (chicken==cute) {
                  keep();
              } else {
                  eat();
              }

* No chickens were harmed in this code example.

The Document Object Model

The Document Object Model, or the DOM for those in the know... is a hierarchical struture consisting of all elements within an html document and provides a framework for JavaScript to access and make use of these elements.

For example, the code below will access the <p> tags within an html document and return an array-like object. 

            
              const elementRefArray = document.getElementsByTagName('p')

Arrays of and Objects of Desire

Arrays are single objects that contain multiple values stored in a list of discrete things. Objects are self contained things that hold data about the thing. For example an arrary could consist of a list of types of luxury cars, whereas the object might contain data about each individual luxury car. 

              
                //An array example
                let luxCars = ["Porsche", "Lamborgini", "Tesla"];
              
            

              
                // An object example
                let Porsche = {
                  color: Space Grey,
                  top speed: 300km/hr,
                  price: $200,000  
                }

Arrays can be accessed by referencing the index (number) corrolated to the desired value, the first being 0 indexed. For example, the code below would access "Porsche". 

            
              console.log(luxCars[0])

However, Objects are accessed by a dot property accessor or square brackets property accessor: 

            
              console.log(Porsche.color)
              console.log(Porsche[color])
